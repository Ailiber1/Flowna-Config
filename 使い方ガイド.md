## <a name="what-is-flowna"></a>🎯 Flowna Configとは？

### 簡単に言うと...

**「思考や作業の流れを、絵で描くように整理できるツール」**です。

```
例えば、こんな流れを可視化できます:

[アイデア] → [企画] → [開発] → [テスト] → [リリース]
    ↓
[フィードバック]
```

### 何ができるの？

✅ **作業の流れを図にできる**
  - 四角いカード（ノード）を配置
  - 矢印（接続線）で繋ぐ
  - 誰が見ても分かる

✅ **自由にカスタマイズ**
  - 名前、色、アイコンを自由に変更
  - あなたの好きなように

✅ **保存して何度も使える**
  - 作った図はフォルダに保存
  - いつでも呼び出せる
  - テンプレートも使える

✅ **外部サービスと連携**
  - GitHubと繋げる
  - AI（Claude）と連携
  - 自動化もできる

### どんな人に向いてる？

- 📝 プロジェクト管理をしたい人
- 🎨 アイデアを整理したい人
- 💼 業務フローを可視化したい人
- 👥 チームで情報共有したい人
- 🔰 **プログラミング経験がなくてもOK！**

## <a name="basic-usage"></a>🚀 基本的な使い方

### 画面の見方

```
┌────────────┬─────────────────────────────────────┐
│            │ [検索] [AI] [設定]                  │ ← 上部バー
│ サイドバー │─────────────────────────────────────│
│            │                                     │
│ フォルダ   │         作業エリア                  │ ← キャンバス
│ 一覧      │     （ここにノードを配置）          │
│            │                                     │
│ コネクタ   │                                     │
│ 一覧      │                                     │
│            │                                     │
│ 言語切替   │ [状態] [ボタン]                     │ ← 下部バー
└────────────┴─────────────────────────────────────┘
```

#### 左側：サイドバー
- **フォルダ**: 作成したワークフローを整理

#### 中央：キャンバス（作業エリア）
- ノードを配置する場所
- 自由にドラッグして動かせる
- ズームイン・アウトできる

#### 上部：検索バー
- ノードを検索できる
- 設定を変更できる

#### 下部：ステータスバー
- 現在の状態を表示
- 保存ボタンなどがある

```
┌─────────────────────────────┐
│   🎉 Welcome to Flowna!     │
│                             │
│   3つの簡単なステップで     │
│   使い方を学びましょう      │
```

**おすすめ**: 初めての方は「始める」を押して、チュートリアルを体験してください。

## <a name="node-operations"></a>📦 ノードの操作

### ノードとは？

**ノード = 作業やプロセスを表す四角いカード**

```
┌─────────────────┐
│ 🤖 データ収集    │ ← これが「ノード」
│ AIで情報を集める │
└─────────────────┘
```

各ノードには：
- **名前**: 自由に付けられる
- **アイコン**: 絵文字や画像
- **色**: 好きな色に変更可能
- **説明**: メモを追加できる

### ノードの追加方法

#### 方法1: ボタンから追加
1. サイドバーの「Add Node」ボタンをクリック
2. ウィンドウが開きます

```
┌─────────────────────────────┐
│ Add Node                    │
├─────────────────────────────┤
│ Node Name *                 │
│ [データ収集]                │ ← 名前を入力
│                             │
│ Icon                        │
│ [🤖] [📊] [💡] [選択]      │ ← アイコンを選ぶ
│                             │
│ Color                       │
│ [●●●●●]                    │ ← 色を選ぶ
│                             │
│ Category                    │
│ [AGENT ▾]                   │ ← カテゴリ選択
│                             │
│ Description                 │
│ [説明を入力...]            │
│                             │
├─────────────────────────────┤
│      [Cancel]  [Create]     │
└─────────────────────────────┘
```

3. 必要な情報を入力
   - **Node Name（必須）**: 「データ収集」など
   - **Icon（任意）**: 🤖 など好きな絵文字
   - **Color（任意）**: 青、緑など好きな色
   - **Category（任意）**: グループ分け
   - **Description（任意）**: 詳しい説明

4. 「Create」ボタンを押す
5. キャンバスの中央にノードが出現！

#### 方法2: 右クリックから追加
1. キャンバスの空いている場所で右クリック
2. 「Add Node」を選択
3. 同じウィンドウが開きます

### ノードの移動方法

**超簡単！ドラッグ＆ドロップするだけ**

1. ノードの上にマウスを置く
2. 左クリックしたまま移動（ドラッグ）
3. 好きな場所で離す（ドロップ）

```
[ノード]  →  ドラッグ  →  [ノード]
  ここから               ここへ移動！
```

**ヒント**: 複数のノードを配置して、分かりやすく整理しましょう。

### ノードの編集方法

#### 方法1: ダブルクリック
1. ノードをダブルクリック
2. 編集ウィンドウが開く
3. 名前や色を変更できる

#### 方法2: 右クリックメニュー
1. ノードを右クリック
2. 「Edit」を選択
3. 編集ウィンドウが開く

### ノードのカスタマイズ

**すべて自由に変更できます！**

```
変更前:
┌─────────────┐
│ AGENT       │ ← デフォルトの名前
│ エージェント │
└─────────────┘

変更後:
┌─────────────┐
│ 🤖          │ ← 好きなアイコン
│ データ収集AI │ ← 自分で付けた名前
└─────────────┘
```

**変更できるもの**:
- ノード名
- 表示名
- アイコン（絵文字または画像）
- 色
- カテゴリ（または新しく作成）
- 説明文

### ノードの削除方法

#### 方法1: Deleteキー
1. ノードをクリックして選択
2. キーボードの「Delete」キーを押す
3. 確認メッセージが表示される
4. 「はい」を選択

#### 方法2: 右クリックメニュー
1. ノードを右クリック
2. 「Delete」を選択
3. 確認メッセージが表示される
4. 「はい」を選択

**注意**: 削除すると、そのノードに繋がっている線も一緒に消えます。

## <a name="connections"></a>🔗 接続の作り方

### 接続線とは？

**ノード同士を繋ぐ矢印のような線**

```
[ノードA] ─────→ [ノードB]
          ↑
       この線が「接続」
```

接続線は：
- **データの流れ**を表す
- **作業の順序**を示す
- **関係性**を可視化する

### 接続の作り方

#### ステップ1: 出力ポートを探す
各ノードの右側に **小さな丸（●）** があります。これが「出力ポート」

```
┌─────────────────┐
│ データ収集      │ ●← ここ！（出力ポート）
└─────────────────┘
```

#### ステップ2: ドラッグ開始
1. 出力ポート（●）にマウスを置く
2. 左クリックしたまま動かす
3. **点線が出現**して、マウスについてくる

```
┌─────────────────┐
│ データ収集      │ ●- - - → マウス
└─────────────────┘
         ↑
    ゴーストライン（点線）
```

#### ステップ3: 入力ポートで離す
1. 繋げたいノードの左側にある丸（入力ポート）まで移動
2. マウスボタンを離す
3. **接続完了！**

```
┌─────────────┐         ┌─────────────┐
│ データ収集  │ ●───→ ● │ データ処理  │
└─────────────┘         └─────────────┘
                ↑
           接続線（緑色）
```

#### 完成！
接続が完成すると：
- **緑色の線**が表示される
- **流れるアニメーション**が見える
- **小さな光の粒**が線の上を移動する

### 接続のキャンセル方法

**間違えた場合**:
- 空いている場所でマウスボタンを離す
- または「Esc」キーを押す
- 接続はキャンセルされます

### 接続の削除方法

#### 方法1: クリック + Deleteキー
1. 削除したい接続線をクリック
2. 線が明るく光る（選択状態）
3. 「Delete」キーを押す
4. 接続が消える

#### 方法2: 右クリックメニュー
1. 接続線を右クリック
2. 「Delete Connection」を選択
3. 接続が消える

## <a name="save-load"></a>💾 ワークフローの保存と読込

### ワークフローとは？

**作成したノードと接続線の全体**を「ワークフロー」と呼びます。

```
ワークフロー = ノード + 接続線 + 配置
```

### 保存方法

#### ステップ1: 保存ボタンをクリック
1. 画面下部の「Save」ボタンをクリック
2. または「Ctrl + S」キーを押す

#### ステップ2: 情報を入力

```
┌─────────────────────────────┐
│ Save Workflow               │
├─────────────────────────────┤
│ Workflow Name *             │
│ [Webアプリ開発フロー]       │ ← 名前を付ける
│                             │
│ Description                 │
│ [フロントエンドからDBまで]  │ ← 説明（任意）
│                             │
│ Select Folder *             │
│ [構成 ▾]                    │ ← フォルダを選ぶ
│                             │
│ Tags                        │
│ [開発, Web, プロジェクト]   │ ← タグ（任意）
│                             │
├─────────────────────────────┤
│      [Cancel]  [Save]       │
└─────────────────────────────┘
```

- **Workflow Name（必須）**: 分かりやすい名前を付ける
- **Description（任意）**: 何のワークフローか説明
- **Select Folder（必須）**: 保存先フォルダを選ぶ
- **Tags（任意）**: 検索用のタグ

#### ステップ3: 保存完了
「Save」ボタンを押すと：
- 画面右上に「保存しました」と表示
- サイドバーのフォルダ内に追加される
- **サムネイル画像**が自動生成される

### 読込方法（開く方法）

#### ステップ1: フォルダを開く
サイドバーの「FOLDERS」セクションを見る

```
FOLDERS
📁 構成
   └─ [サムネイル] Webアプリ開発フロー
   └─ [サムネイル] データ処理
📁 接続
```

#### ステップ2: ワークフローをクリック
1. 開きたいワークフローをクリック
2. 未保存の変更がある場合、確認メッセージ
3. ワークフローが読み込まれる
4. ノードがフワッと表示される

**簡単！**

### 複数のワークフローを管理

**いくつでも作成できます**:
- プロジェクトごとに分ける
- 用途ごとに分ける
- 日付ごとに分ける

## <a name="folders"></a>📁 フォルダで整理する

### フォルダの役割

**ワークフローをグループ分けして整理**

```
📁 構成
   ├─ Webアプリ開発
   ├─ モバイルアプリ
   └─ インフラ構築

📁 接続
   ├─ API連携
   └─ データベース設計
```

### デフォルトのフォルダ

初回起動時、以下のフォルダが自動で作成されます：

- 📁 **構成**: システム構成用
- 📁 **接続**: 連携・接続用
- 📁 **モジュール**: 機能単位用
- 📁 **Data Input**: データ入力用
- 📁 **ユーザード**: ユーザー関連用
- 📁 **オーナー**: 管理者用

**これらは例です。自由に変更・削除できます。**

### 新しいフォルダの作成

#### 方法1: +ボタン
1. 「FOLDERS」の横の「+」ボタンをクリック
2. フォルダ名を入力
3. Enterキーで確定

#### 方法2: 右クリック
1. フォルダエリアで右クリック
2. 「New Folder」を選択
3. フォルダ名を入力

### フォルダの操作

#### 名前を変える
1. フォルダを右クリック
2. 「Rename」を選択
3. 新しい名前を入力

#### アイコンを変える
1. フォルダを右クリック
2. 「Change Icon」を選択
3. 好きな絵文字を選ぶ

#### 削除する
1. フォルダを右クリック
2. 「Delete」を選択
3. 中身があれば警告が出る
4. 確認して削除

### ワークフローを移動

**ドラッグ＆ドロップで簡単移動**:
1. ワークフローをクリック
2. そのまま別のフォルダまでドラッグ
3. フォルダが光る
4. そこで離すと移動完了

## <a name="connectors"></a>🔌 コネクタで外部サービスと連携

### コネクタとは？

**外部のサービスとFlownaを繋ぐプラグイン**

```
Flowna ←→ コネクタ ←→ 外部サービス
                      (GitHub, Claude等)
```

### 使えるコネクタ

サイドバーの「CONNECTORS」セクションに表示されます：

```
CONNECTORS         [+]
🐙 GitHub          ● ← 緑=接続済み
⚡ Claude Code     ○ ← グレー=未接続
☁️ Firebase        ●
💎 Gemini Code     ○
🔗 Custom API      ○
```

#### 1. GitHub 🐙
**できること**:
- コードのリポジトリと連携
- GitHub Actionsを実行
- 自動テスト・デプロイ
- Issueの管理

**こんな人に**: プログラマー、開発チーム

#### 2. Claude Code ⚡
**できること**:
- AIにコード生成を依頼
- コードレビュー
- ドキュメント作成
- ノードの自動生成

**こんな人に**: コーディング支援が欲しい人

#### 3. Firebase ☁️
**できること**:
- データベース連携
- ユーザー認証
- 自動デプロイ
- サーバーレス実行

**こんな人に**: Webアプリを作る人

#### 4. Gemini Code 💎
**できること**:
- AIコード補完
- リファクタリング提案
- セキュリティチェック

**こんな人に**: コード品質を上げたい人

#### 5. Custom API 🔗
**できること**:
- 任意のAPIと連携
- カスタム設定
- 自由な連携

**こんな人に**: 独自システムがある人

### コネクタの追加方法

#### ステップ1: コネクタを選ぶ
1. 「CONNECTORS」の「+」ボタンをクリック
2. または未接続のコネクタをクリック

#### ステップ2: 認証設定

```
┌─────────────────────────────┐
│ GitHub Connector            │
├─────────────────────────────┤
│ API Token *                 │
│ [ghp_xxxxxxxxxxxx]          │ ← トークンを入力
│                             │
│ Repository                  │
│ [username/repo-name]        │
│                             │
│ [Test Connection]           │ ← テスト
│                             │
├─────────────────────────────┤
│      [Cancel]  [Connect]    │
└─────────────────────────────┘
```

1. 必要な情報を入力（API Keyなど）
2. 「Test Connection」で接続テスト
3. 成功したら「Connect」をクリック

#### ステップ3: 接続完了
- ステータスドットが緑色に変わる ●
- 「接続しました」と表示される
- すぐに使い始められる

### コネクタの使い方

#### ノードから使う
1. ノードを右クリック
2. 「Link to Service」を選択
3. コネクタを選ぶ
4. リソース（リポジトリなど）を選択
5. ノードにアイコンが付く

#### データをインポート
1. コネクタを右クリック
2. 「Import Data」を選択
3. 取り込みたいデータを選ぶ
4. 自動でノードが作成される

**例**: GitHubのIssueを取り込むと、Issue1つにつき1ノード作成

## <a name="advanced-features"></a>🎁 便利な機能

### テンプレート機能

**よく使うパターンをすぐに使える**

#### テンプレートの使い方
1. 新規ワークフロー作成時
2. 「テンプレートから作成」を選ぶ

```
┌─────────────────────────────┐
│ ⚪ 空のワークフロー           │
│                             │
│ 📋 テンプレート:            │
│ ・💻 Webアプリ開発フロー     │
│ ・📊 データ処理             │
│ ・✅ 承認フロー             │
│ ・📝 タスク管理             │
└─────────────────────────────┘
```

3. テンプレートを選択
4. すでにノードと接続が配置されている
5. 自由にカスタマイズできる

#### 用意されているテンプレート

**1. Webアプリ開発フロー 💻**
```
[ユーザー入力] → [フロントエンド] → [API] 
    → [データベース] → [結果表示]
```
用途: Webアプリの基本構成

**2. データ処理パイプライン 📊**
```
[データ収集] → [クリーニング] → [変換]
    → [分析] → [保存] → [可視化]
```
用途: データ分析作業

**3. 承認フロー ✅**
```
[申請] → [確認] → [承認] → [実行]
```
用途: 社内承認プロセス

**4. シンプルタスク管理 📝**
```
[未着手] → [作業中] → [完了]
```
用途: 個人タスク管理

**5. CI/CDパイプライン 🚀**
```
[コード変更] → [テスト] → [ビルド] 
    → [デプロイ] → [監視]
```
用途: 開発の自動化

### 検索機能

**作ったノードをすぐに見つける**

#### 使い方
1. 画面上部の検索ボックスをクリック
2. または「Ctrl + F」を押す
3. ノード名を入力
4. ヒットしたノードが黄色くハイライト
5. カメラがそのノードに移動

**検索できるもの**:
- ノード名
- 説明文
- メモ
- カテゴリ名

### メモ機能

**ノードに詳しい情報を追加**

#### 使い方
1. ノードを右クリック
2. 「Memo」を選択
3. メモを入力

```
┌─────────────────────────────┐
│ Memo                        │
├─────────────────────────────┤
│ [このノードは〇〇を処理する  │
│  担当: 田中さん             │
│  期限: 2024/02/01]          │
├─────────────────────────────┤
│      [Cancel]  [Save]       │
└─────────────────────────────┘
```

4. 「Save」で保存
5. ノード右上に📝アイコンが表示される
6. アイコンにマウスを置くとメモが見える

### ステータス機能

**作業の進捗を管理**

#### 使い方
1. ノードを右クリック
2. 「Set Status」を選択
3. ステータスを選ぶ:
   - **Todo（未着手）**: グレーのドット
   - **Doing（作業中）**: 黄色のドット（点滅）
   - **Done（完了）**: 緑のドット

**一目で進捗が分かる！**

### ズームとパン

#### ズーム（拡大・縮小）
- **マウスホイール**: 上で拡大、下で縮小
- **Ctrl + プラス**: 拡大
- **Ctrl + マイナス**: 縮小

#### パン（移動）
- **キャンバスをドラッグ**: 画面全体を移動
- **スペース + ドラッグ**: 一時的にパンモード

#### フィット表示
- **Ctrl + 0**: 全ノードが画面に収まる

### エクスポート・インポート

#### エクスポート（出力）
1. 「Export」ボタンをクリック
2. JSON形式でダウンロード
3. バックアップやシェアに使える

#### インポート（読込）
1. 「Import」ボタンをクリック
2. JSONファイルを選択
3. ワークフローが復元される

## <a name="faq"></a>❓ よくある質問

### Q1: プログラミングの知識は必要？
**A: 不要です！** マウス操作だけで使えます。ドラッグ＆ドロップができれば大丈夫。

### Q2: 作ったワークフローは他の人と共有できる？
**A: できます！** 
- エクスポート機能でJSONファイルを出力
- そのファイルを相手に送る
- 相手がインポートすれば同じワークフローが見られる

### Q3: 無料で使える？
**A: 基本機能は無料です。**（実際の料金体系は運営による）

### Q4: モバイル（スマホ・タブレット）でも使える？
**A: PC推奨ですが、タブレットなら使えます。** スマホは画面が小さいため少し難しいです。

### Q5: データはどこに保存される？
**A: 2箇所に保存されます:**
- **ローカル**: あなたのPC内（localStorage）
- **クラウド**: Firebase（オンライン時）

### Q6: オフラインでも使える？
**A: 使えます！** ローカルに保存されるので、インターネットなしでも大丈夫。オンライン時に自動同期されます。

### Q7: ノードはいくつまで作れる？
**A: 制限なし！** ただし、100個を超えると少し重くなるかもしれません。

### Q8: 間違えて削除してしまった！
**A: まだ保存していなければ「Ctrl + Z」で戻せます。**（元に戻す機能は次期実装予定）
保存後は復元できないので、大事なものはバックアップを。

### Q9: GitHubとか、専門的なツールは使わなくてもいい？
**A: もちろん！** コネクタは使いたい人だけが使う機能です。使わなくても十分活用できます。

### Q10: 他のツール（Miro、Figmaなど）との違いは？
**A: Flownaは「ワークフローの可視化」に特化しています:**
- ノードと接続線に特化
- 外部サービス連携が簡単
- CI/CDとの統合
- プログラマーにも非プログラマーにも優しい

### Q11: 英語が苦手なんだけど...
**A: 日本語で使えます！** 
- サイドバー下部の「EN / JP」で切り替え
- すべてのUIが日本語に

### Q12: チュートリアルをもう一度見たい
**A: 設定メニューから見られます:**
1. 右上の設定アイコン⚙️をクリック
2. 「Show Tutorial」を選択
3. チュートリアルが再表示

### Q13: バグを見つけた、機能リクエストしたい
**A: フィードバック機能を使ってください:**
1. 設定メニュー
2. 「Send Feedback」
3. 内容を入力して送信
